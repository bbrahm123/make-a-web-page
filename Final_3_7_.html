DataFlow

The following descibes how data moves from the users desktop through the internet back:

  • The user opens the browser, a WEB page is displayed. All the text on the page is written in HTML.

  • The user types a URL into a browser.

  • The browser reads the URL and translates it into an HTTP request.

  • If the computer has a working physical connection to the internet, the HTTP request will be forwarded
    to the "cloud".

  • The request searches various sights looking for its URL destination.The destination will be located on
    a server configured specifically to maintain data accessible from the users across the internet.

  • The destination server is usually stored with other similar type servers in a closest or data center.

TAGS and Elements

The following is a brief description( well, my interpretation of) how tags and text become elements.

  • The Syntax of HTML are TAGS embedded in the WEB page, next to the text. the Tags tell the browser how
    to configure the text

  • The browser, using the Document Object Model, turns HTML tags and text into elements that form a nested
    tree-like structure.

  • The element is created with opening and closing brackets, the content section between them, as in the
    following:

    • <em>this is for emphasis<em>


  • If the content section is empty and only the location of a source file is listed, the closing tag does
    not require the / slash. This type of closing tag is a "void tag". If a closing tag is not used, the browser
    may fall into an unsteady state.

Lesson 2: Creating a Structured Document with HTML

HTML, TAGS and Trees and the structure of the WEB page

HTML is a language which uses TAGS embedded in the WEB page, which communicate to the browser, information about the structure of the WEBpage. The browser, utilizing the Document Object Model, turns HTML tags into elements that form a nested tree-like structure.
  • An element is created from two tags and content:

    <start tag> content <end tag>>

  • The content is text or images, or video or whatever you want.

  • Each tag is a name or type that defines the kind of element that will be created. The <p> tag identifies text blocked as a paragraph.The <em> italicizes text.TAGS also provide information such as links to other documents or images.

Boxes

  • All elements on a WEB page are a box container; The elment might look like a circle, but its a rectangle containing the circle. The size and shape of the circle can be changed by adjusting the circles pixel number in the Style section of Developer Tools within the browser. Looking at the WEB page as a series of boxes allows the the designer to easily move elements on the page and experiment with different layout options, and hopefully creating a design the customer wants.

Text Editors

  • When writing code, programmers use special text editors (like Sublime Text for example). These editors make the programmer's life easier. For example, some text editors will automatically generate a closing HTML tag when you write an opening tag.

Lesson 3 -Adding CSS Style to HTML Structure

  • Cascading Style Sheets is a programming language used for controlling the visual style of the web page. CSS uses specific syntax and rules to control how elements such as the size of the font, background color and borders look on a page. To access and change the visual stye of the Webpage you need to access the Developer Tools within client browser. For example, you can say all the should have a specific color such as red or blue.

The Selector

  • The Selector tells the browser where to apply the rule in the name value-pair. The syntax for a selector is configured as follows. The selector, curly bracies, the property name such as "background color", and the value such as "red".( hex RGB format). You can have multipe value pairs on each rule

     selector {
     propertyname:value;
}

  • there are mulitple types of selectors, for example a class selector will find elements with a matching class attributes.

     .class {
     propertyname:value;
}

Cascade

  • There are three methods available to include CSS Styling in the webpage

    1. Write CSS in the Head section of the HTML document.

    2. Link your HTML document to a separate CSS file - the link tag is added in the head section.

    3. Write your style within the body of the HTML document- not a good idea, results in a lot of repeated code.


  • Styles applied to HTML elements can come form multile sources, multiple link tags can be inserted in the head element,linking multiples CSS pages. You can also use a combination of link tags and embedded styles.

Inheritance

  • Inheritance allows some CSS properties applied to an element to flow down to nested or child elements within the HTML DOM tree structure. A font-family property will, by default, be applied to a HTML element, and therefore all the elements nested within that body.Fortuneatly not all CSS values can be inherited, for example a border cannot be inherited; multiple nested borders would look pretty strange!

  • Inheritable styles include font specfic items, such font-sizes, font-styles, etc. Also inheritable are List related styles, such as list-style type.

  • Non-inheritable styles include, margins, padding and border

Box Properties

Every element in HTML is a box of a certain number of pixels. and the designer can adjust the following 4 properties of this box with CSS.

  • The content- the center, the focus of the page.

  • The Border- most borders are not divisible by default, but the user can control the width, style and color of the border.

  • The Padding- the padding adds space between the border of a box and the content of a box. Padding pushes content in, away from the border.

  • The Margin- the space between the box any adjacent elements; the space between the boxes.

Box Sizing and Positioning

By default, size of each element is equal to the sum of the border, padding and content widths of the element. The designer can adjust each of these values, on 4 sides of the box. Because of this flexibility it can often be hard to get the size of a box just right. There are several techniques you can use to help deal with sizing issues among them are

     *{ 
	box-sizing; border-box;
	}

  • this makes the calculation element size include the border and padding so its easier to make layouts. You only need to set the width of the overall box.

  • pixels - change the content page from a set pixel size to percent will effect the screen size. If set to a pixel size the screen image will remain static screen itself is changed.

  • Max Width - of the box, it will resize when necessary, never larger than the Max.

  • Flexible Box layout - provides an efficient way to layout a line and distribute space among items in a container. If you want several elements to be next to each other, you can change the dispaly attribute of the parent container to value "flex".

  • To ensure that versions of the browsers understand new elements, you need to insert browser prefexs into the HTML code. The prefixes for the most common browsers are listed below

MS for Microsoft Explorer
MOS for Firefox
